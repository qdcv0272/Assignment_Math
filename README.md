# 달빛약속 - DragEditor (프로토타입)

## 간단 소개

이 저장소는 초등 수준의 코드 조립형 문제를 만들기 위한 에디터 프로토타입입니다. 사용자는 문법 조각(토큰)을 드래그하여 왼쪽 코드 영역에 조립하고, 실행 버튼으로 결과를 확인합니다. 본 버전은 다음과 같은 UX·비즈니스 규칙을 반영합니다:

- 정답 판정은 반드시 `정답` 변수에 숫자를 할당하고 `정답 보여주기`(또는 출력하기)를 수행해야 인정됩니다.
- 줄바꿈은 드래그 가능한 조각으로 복제되지 않고, 우측 패널의 **줄바꿈 버튼**으로만 삽입됩니다.
- 조립된 토큰은 클릭(또는 Enter/Space)으로 제거되며 사용 가능한 조각 목록 맨 뒤로 돌아갑니다.

## 핵심 기능

- 엄격한 정오답 검사: `정답` 할당 + `보여주기` 출력 검증 (차이시 인라인 에러 표시)
- 줄바꿈 버튼(버튼식 삽입) 및 드래그 방지
- 토큰 클릭/키보드 제거(접근성 고려)
- `BottomActions`를 본문(`.page-body`) 안으로 이동하여 스크롤 안정화
- UI 스타일 정리(배너/패널 색상, 버튼 가독성)

## 기술 스택

- React + TypeScript
- Vite (개발 서버)
- Zustand (상태관리) — `src/stores/useDragEditorStore.ts`
- `@dalbit-yaksok/core` (실행/샌드박스 세션)
- CSS (프로젝트 전반 스타일은 `src/css/`) Panda 는 추후 적용

## 주요 파일/구조

- `src/components/DragEditor.tsx` — 에디터 메인 로직 (미리보기 생성, 실행 흐름, 정오답 검사)
- `src/components/Editor/CodeArea.tsx` — 코드 영역 렌더링 및 토큰 상호작용
- `src/components/Editor/GrammarPanel.tsx` — 우측 문법 패널 + 줄바꿈 버튼
- `src/stores/useDragEditorStore.ts` — 상태 관리 (코드 조각, 사용 가능 조각, 출력, 에러 등)
- `src/css/DragEditor.css`, `src/styles.css` — 스타일 및 테마 변수

## 주요 동작 설명

- 미리보기 생성: `formatPreview(tokens)`로 조각들을 라인 단위 텍스트로 변환하여 실행 소스를 만듭니다.
- 실행: `YaksokSession`을 통해 모듈을 추가하고 실행하며, `stdout`/`stderr`를 받아 `output`/`error`로 보여줍니다.
- 정오답 검사: `getPrintedInfo(source)`로 예상 출력 변수와 `정답` 존재 유무를 계산하고, 실제 실행 출력과 비교합니다.
- 줄바꿈 규칙: `줄바꿈` 토큰은 드래그 목록에서 필터링되어 생성되지 않으며, 버튼 클릭 시 `insertLineBreak()`로 코드에 추가됩니다.

## 개발자 노트

- 로그: 개발 중 사용한 콘솔 로그를 한글화하고 불필요한 로그는 제거했습니다. 필요 시 다시 활성화 가능합니다.
- 훅 규칙: `useDragEditorStore` 훅 사용 위치를 고쳐 React 훅 규칙 위반을 해결했습니다.
- 리팩토링: `handleStart`를 `runSession` 등으로 분리해 가독성을 개선했습니다.

## 테스트 및 검증 체크리스트

1. 오른쪽 문법 칩을 드래그하여 왼쪽 코드 영역에 조립
2. `정답` 변수에 숫자 할당하지 않으면 에러가 표시되는지 확인
3. `정답` 할당 후 `보여주기`로 출력하지 않으면 에러 표시 확인
4. 정답 출력 값과 기대값이 다른 경우 에러 표시 확인
5. 정답이 맞아도 사용하지 않은 조각이 남아 있으면 경고(현재 `alert`) 표시 확인
6. 줄바꿈 버튼 클릭 시 왼쪽에 줄바꿈이 추가되고 오른쪽에 줄바꿈 칩이 생성되지 않는지 확인

## 향후 개선 제안

- 남은 조각 경고를 `alert`에서 디자인 일관된 커스텀 모달로 전환
- 토큰 추가/제거 애니메이션(GSAP) 보강
- 유닛 테스트(vitest)로 `getPrintedInfo` 등 핵심 로직 자동화
