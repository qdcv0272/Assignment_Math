# 03. 에이전트 협업 기록 — 문제 추출

작성일: 2026-04-23

요약: 교재·문서에서 문제(예: 초등수학 문제)를 추출하고 구조화하는 작업에서 에이전트를 활용한 방법을 기록합니다.

## 사용한 에이전트 도구

- 텍스트 분석 및 파싱 제안: 대화형 에이전트
- 코드·스크립트 생성: `apply_patch`로 추출 스크립트 추가
- 검색·검토: `semantic_search`, `read_file`
- 작업 추적: `manage_todo_list`

## 에이전트에게 위임한 작업의 범위와 판단 근거

- 위임한 작업:
  - 원문 텍스트에서 문제·정답·난이도 등 필드 추출 규칙 초안 작성
  - 간단한 파싱 스크립트(정규식/경계 기반) 초안 생성
  - 추출한 결과를 JSON/CSV로 직렬화하는 포맷 제안
- 판단 근거:
  - 대량 텍스트에서 규칙 기반 추출은 반복성이 높아 자동화가 유리
  - 복잡한 자연어 이해가 필요한 경우에는 사람이 샘플을 검증

## 에이전트의 산출물을 검증한 방법

- 샘플셋에 대해 수동 검토(랜덤 샘플 50~100건)로 정밀도/재현율 확인
- 엣지케이스(그림·표 포함 문제 등)에 대해 실패율 분석
- 결과 파일을 로드해 소비자(프론트엔드/DB)에 정상적으로 적재되는지 확인

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 실패 사례를 모아 룰(정규식·파서) 보완 지시
- 샘플 기반 피드백 루프: 잘못 추출된 사례를 에이전트에게 맥락으로 제공해 재추출
- 필요 시 사람이 직접 정제 후 룰로 일반화
