# 03. 에이전트 협업 기록 — UI 컴포넌트 개발

작성일: 2026-04-23

요약: `TokenEditor`, 페이지 컴포넌트 등 UI 요소를 에이전트와 협업해 설계·구현·검증한 기록입니다.

## 사용한 에이전트 도구

- 코드 생성 및 패치: `apply_patch`
- 코드 참조: `read_file`, `grep_search`
- 실행·검증: `run_in_terminal` (빌드/테스트), 브라우저 수동 확인
- 작업 추적: `manage_todo_list`

## 에이전트에게 위임한 작업의 범위와 판단 근거

- 위임한 작업:
  - 컴포넌트 스켈레톤 및 props 타입 제안
  - 반복적 UI 패턴(폼, 리스트, 버튼 등) 구현 초안
  - 유닛 테스트·스토리(간단한 예시) 스켈레톤 작성
- 판단 근거:
  - UI 스켈레톤·패턴 코드는 규칙적이며 빠르게 제안하고 반복 수정하기 쉬움
  - 인터랙션·접근성 검증은 사람이 수동 확인 필요

## 에이전트의 산출물을 검증한 방법

- 렌더링 확인: 주요 화면 렌더링 및 사용자 흐름 수동 점검
- 타입/린트/유닛 테스트 실행으로 기본 동작 보장
- 접근성(A11y) 체크리스트로 키보드·스크린리더 동작 검토

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 사용자 흐름 중심의 재요구: 기대 동작(예: 포커스 흐름)을 시나리오로 제공
- 실패 케이스를 명시한 테스트를 추가해 재현 가능한 피드백 루프 생성
- UI 규칙(디자인 토큰, 클래스 네이밍)을 명확히 고정해 재발 방지
