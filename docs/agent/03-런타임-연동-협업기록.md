# 03. 에이전트 협업 기록 — 런타임 연동

작성일: 2026-04-23

요약: 개발 서버, 빌드 파이프라인, 환경변수 연동 등 런타임 관련 설정을 에이전트와 협업한 기록입니다.

## 사용한 에이전트 도구

- 컨텍스트 읽기: `read_file`, `grep_search`
- 구성제안 및 패치: `apply_patch`
- 검증 실행: `run_in_terminal` (로컬 빌드/테스트 실행)
- 작업 추적: `manage_todo_list`

## 에이전트에게 위임한 작업의 범위와 판단 근거

- 위임한 작업:
  - `vite.config.ts`, `tsconfig.json`, `package.json` 스크립트 제안·초안
  - 환경변수 패턴(.env) 설계 초안
  - 간단한 런타임 문제(빌드 오류 추적용 로그 해석) 우선 진단
- 판단 근거:
  - 설정은 패턴 기반이며, 표준 관행을 따라 제안할 수 있음
  - 실제 런타임 검증은 사람이 빌드/테스트로 확인해야 함

## 에이전트의 산출물을 검증한 방법

- 로컬 빌드 및 dev 서버 실행: `npm run dev` / `pnpm dev`로 정상 시작 확인
- 타입 체크 및 린트 실행으로 구성 오류 확인
- 변경된 설정의 영향 범위를 수동으로 탐색(주요 페이지 접근 등)

## 에이전트가 잘못된 방향을 제시했을 때 교정 방법

- 로그·오류 메시지를 함께 제공해 원인 기반 재질문
- 제안된 설정을 단계별로 적용해 문제의 단계 구분(롤백 포인트 확보)
- 보안·배포 요구사항이 있는 경우 사람 검토를 필수화
